JDBC之PreparedStatement

我们先来看一个例子：

 public static boolean login(String id,String password)throws Exception{
        Class.forName("com.mysql.jdbc.Driver");
        Connection con = DriverManager.getConnection("jdbc:mysql://localhost:3308/bhzc","root","mysql");
        Statement stat = con.createStatement();
        String sql = "select * from student where id = '"+ id + "' and psw = '" + password + "'" ;
        System.out.println(sql);
        ResultSet rs = stat.executeQuery(sql);
        return rs.next();
    }

这里简单的写了一个登录的方法，用户登录的时候查询账号和密码，如果查询返回的结果集里有数据，那么返回登录成功。

下面是用户输入：

public static void main(String[] args)throws Exception {
        // 这是sql语句 select * from student where id = '123456' and psw = '12345'
        String id = "123456";
        String password = "12345";
        boolean b = login(id,password);
        System.out.println(b);
    }

这个正常返回的hitrue，因为数据库中有这条数据，但是我们看下面的输入:

String id = "a' or 'a' = 'a";
String psw = "a' or 'a' = 'a";

注意这两个传入的参数 a' or 'a' = 'a 把要判断的语句变成了'a' or 'a' = 'a'。这时候这个参数的sql语句就变成了 select * from student where id = 'a' or 'a' = 'a' and psw = 'a' or 'a' = 'a'。对比上面的sql语句我们可以发现，这条SQL语句向数据库中添加了一个数据库中的关键字 or ，强行输入sql语句的片段，后面的条件成立结果就为true（这就是sql注入）但是PreparedStatement可以防止这种sql注入！！

PreparedStatement接口继承Statement，并与之在两方面有所不同：

PreparedStatement 实例包含已编译的 SQL 语句。这就是使语句“准备好”。包含于 PreparedStatement 对象中的 SQL 语句可具有一个或多个 IN 参数。IN参数的值在 SQL 语句创建时未被指定。相反的，该语句为每个 IN 参数保留一个问号（“？”）作为占位符。每个问号的值必须在该语句执行之前，通过适当的setXXX 方法来提供。

由于 PreparedStatement 对象已预编译过，所以其执行速度要快于 Statement 对象。因此，多次执行的 SQL 语句经常创建为 PreparedStatement 对象，以提高效率。

作为 Statement 的子类，PreparedStatement 继承了 Statement 的所有功能。另外它还添加了一整套方法，用于设置发送给数据库以取代 IN 参数占位符的值。同时，三种方法 execute、 executeQuery 和 executeUpdate 已被更改以使之不再需要参数。这些方法的 Statement 形式（接受 SQL 语句参数的形式）不应该用于 PreparedStatement 对象。

preparedStatement具体如何防止sql注入的请看这里：https://blog.csdn.net/lisehouniao/article/details/51523497以及https://blog.csdn.net/maxracer/article/details/6118036

下面我们来进行实际操作一下：

public static boolean login_1(String id,String password)throws Exception{
        Class.forName("com.mysql.jdbc.Driver");
        Connection con = DriverManager.getConnection("jdbc:mysql://localhost:3308/bhzc","root","mysql");
        //给出参数模板 所有的值用 ? 替代
        String sql = "select * from student where id = ? and psw = ?" ;
        //调用Connection方法获取PreparedStatement
        PreparedStatement ps = con.prepareStatement(sql);
        //调用PreparedStatement的setXXX系列方法给sql模板中的?赋值
        ps.setString(1,id);//给第一个问号赋值
        ps.setString(2,password);
        ResultSet rs = ps.executeQuery();
        return rs.next();
    }
public static void main(String[] args)throws Exception {
        String id = "123456";
        String password = "12345";

        String id_1 = "a' or 'a' = 'a";
        String psw = "a' or 'a' = 'a";
        boolean b = login_1(id,password);
        System.out.println(b);
    }

这时我们再输入下面的那个例子时候就不会返回true了，这里不输出SQL语句的原因是，PreparedStatement会对传进来的SQL语句进行预编译输出只能输出select * from student where id = ? and psw = ?但是我们在链接数据库执行的时候，执行的它预编译好的SQL语句了。
