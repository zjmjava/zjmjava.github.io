JDBC结果集滚动和元数据

ResultSet之滚动结果集　

　ResultSet表示结果集，它是一个二维的表格。ResultSet内部维护一个行光标（游标），ResultSet提供了一系列的方法来移动游标：

  • void beforeFirst():把光标放到第一行的前面，这也是光标默认的位置；

  • void afterLast()：把光标放到最后一行的后面；

  • boolean first(): 把光标放到第一行的位置上，返回值表示调控光标是否成功；

  • boolean last()：把光标放到最后一行的位置上；

  • boolean isBeforeFirst（）：当前光标位置是否在第一行前面

  • boolean isAfterLast（）：当前光标位置是否在最后一行后面

  • boolean isFirst（）：当前光标位置是否在第一行上

  • boolean isLast（）：当前光标位置是否在最后一行上

  • boolean previous(): 把光标向上挪一行

  • boolean next(): 把光标向下挪一行

  • boolean relative（int row）：相对位移，当row为正数时，表示向下移动row行，为负数时表示向上移动row行。

  • boolean absolute（int row）:绝对位移，把光标移动到指定的行上；

  • int getRow():返回当前光标所有行。

​ 默认的 ResultSet 对象不可更新，仅有一个向前移动的光标。因此，只能迭代它一次，并且只能按从第一行到最后一行的顺序进行。

​ 结果集是否支持滚动，要从Conntection类的createStatement（int,int）方法说起。也就是说创建的Statement决定了使用Statement创建的ResultSet是否支持滚动和/或可更新的对象。

  • 第一个参数的可选值：
    • ResultSet.TYPE_FORWARD_ONLY: 不滚动结果集；（默认值）
    • ResultSet.TYPE_SCROLL INSENSITIVE: 滚动结果集，但结果集数据不会再跟随数据库而变化；
    • ResultSet.TYPE_SCROLL_SENSITIVE: 滚动结果集,但结果集数据不会再跟随数据库而变化；（无数据驱动支持 ）
  • 第二个参数的可选值 ：
    • CONCUR_READ_ONLY:结果集是只读的，不能通过修改结果集而反向影响数据库；
    • CONCUR_UPDATABLE:结果集是可更新的，对结果集的更新可以反向影响数据库。

MySQL的 con.createStatement();默认生成的是可滚动的，其他的数据库暂不支持

ResultSet获取元数据

rs.getMetaData()可以获取结果集元数据，通过ResultSetMetaData的一系列方法可以得到结果集的结构信息，比如字段数、字段名等。 

        Class.forName("com.mysql.jdbc.Driver");
        Connection con = null;
        ResultSet rs = null;
        Statement stmt = null;
        try{
            con = DriverManager.getConnection("jdbc:mysql://localhost:3308/goods","root","mysql");
            //定义stmt执行后获取的ResultSet为可滚动 可更新的结果集
            stmt = con.createStatement(ResultSet.TYPE_SCROLL_INSENSITIVE,ResultSet.CONCUR_UPDATABLE);
            rs = stmt.executeQuery("select * from good");
            int count = rs.getMetaData().getColumnCount();              //获取一共有多少列
            for(int i = 1;i <= count ;i++){
                // rs.getMetaData() 获取结果集元数据
                System.out.print(rs.getMetaData().getColumnName(i));    //输出每个列的列名
                if(i < count) System.out.print("       ");
            }
            System.out.println();
            // 简单的尝试一下对结果集进行修改并影响数据库
            // 具体参照：https://blog.csdn.net/weixin_34081595/article/details/91894626
            // 更新结果集并修改数据库中的数据
           rs.absolute(2);                               //绝对定位，定位到第二行
           rs.updateString("type","笔记本");   //使用列名修改
           rs.updateRow();
            // 向数据库中插入数据
            rs.moveToInsertRow();                            //移动到可插入的行
            rs.updateString(1,"51094");      //更新语句是可以使用列号，也可以使用列名
            rs.updateString(2,"我的笔记本");
            rs.updateString(3,"666");
            rs.updateString(4,"1");
            rs.updateString(5,"杂牌子");
            rs.updateString(6,"笔记本");
            rs.insertRow();                             //插入行
            rs.beforeFirst();                           //将指针返回到最开始的位置，否则遍历时会丢失数据
            while(rs.next()){                           //遍历行
                for (int i = 1; i <= count ; i++) {     //遍历列
                    System.out.print(rs.getString(i));
                    if (i < count) System.out.print("     ");
                }
                System.out.println();
            }
        }catch(Exception e){
            throw new RuntimeException(e);
        }
